# Voice Input Method · 语音输入法

一款基于 Web Speech API 与 Node.js 的全栈语音输入法产品，旨在帮助用户显著提升文本输入效率。

> 设计目标：在 **准确度 / 易用性 / 响应速度 / 成本** 之间取得平衡，提供开箱即用的浏览器内语音输入体验。

## 项目亮点

| 维度 | 解决方案 |
| --- | --- |
| **准确度** | 浏览器原生 ASR + 服务端文本后处理（标点修正、口头禅过滤、数字归一化） + 用户热词词库 |
| **响应速度** | 流式实时识别（interim results），首字延迟 < 300ms；后处理走批量异步 |
| **易用性** | 一键录音、语音命令编辑、热词管理、历史记录、深浅色主题、全键盘可访问 |
| **成本** | 核心 ASR 走浏览器免费 API，后端无状态 + 本地 JSON 存储，可零成本部署 |
| **隐私** | 音频不离开浏览器；只有最终文本经过服务端后处理，可完全本地化部署 |

## 核心功能

- **实时语音转文字** — 边说边出字，支持中英文混合
- **智能后处理** — 自动标点、数字归一化（"二零二六" → "2026"）、过滤口头禅（"嗯、那个、然后"）
- **语音命令** — "删除"、"换行"、"句号"、"清空"、"全选"等编辑指令
- **热词词库** — 用户自定义专业术语，识别时优先匹配
- **历史记录** — 所有最终转写自动保存，可一键复制 / 导出
- **可访问性** — ARIA 标签、键盘快捷键、明暗主题切换

## 快速开始

环境要求：Node.js ≥ 18、Chrome / Edge（Web Speech API 兼容性最佳）。

```bash
# 1. 安装后端依赖
cd backend && npm install

# 2. 启动后端（默认 :3001）
npm start

# 3. 启动前端（任意静态服务器，默认 :3000）
cd ../frontend && npx http-server -p 3000 -c-1
# 或：python3 -m http.server 3000
```

打开 `http://localhost:3000`，点击麦克风按钮开始说话。
